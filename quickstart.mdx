title: "Get started with Dalamud plugin development"
description: "Create your first Dalamud plugin from the official template, implement IDalamudPlugin, inject services, and load the plugin in-game."
---

This guide walks you through everything you need to write, build, and load a Dalamud plugin from scratch. By the end you will have a working plugin that registers a chat command and prints a message in-game.

<Steps>
  <Step title="Install prerequisites">
    Before you write any code, make sure the following tools are installed:

    - **XIVLauncher** — required to inject Dalamud into the game. Download it from [goatcorp.github.io](https://goatcorp.github.io/).
    - **.NET 10 SDK** — Dalamud targets `net10.0-windows`. Download from [dot.net](https://dotnet.microsoft.com/download).
    - **Visual Studio 2022** (Windows) or **JetBrains Rider** — both have first-class C# support. VS Code with the C# Dev Kit extension also works.

    Verify your SDK version:

    ```bash
    dotnet --version
    # Should print 10.x.x or higher
    ```
  </Step>

  <Step title="Create a plugin project from the template">
    The official starting point for new plugins is the [SamplePlugin](https://github.com/goatcorp/SamplePlugin) template repository on GitHub.

    1. Go to [github.com/goatcorp/SamplePlugin](https://github.com/goatcorp/SamplePlugin).
    2. Click **Use this template** → **Create a new repository**.
    3. Clone your new repository locally:

    ```bash
    git clone https://github.com/YOUR_USERNAME/YOUR_PLUGIN_NAME
    cd YOUR_PLUGIN_NAME
    ```

    The template includes a working `.csproj`, a `Plugin.cs` entry point, and the correct NuGet references to Dalamud's API surface. Open the solution in your IDE and restore packages before continuing.
  </Step>

  <Step title="Implement IDalamudPlugin">
    Every Dalamud plugin must implement `IDalamudPlugin`, which extends `IDisposable`:

    ```csharp
    // From Dalamud/Plugin/IDalamudPlugin.cs
    public interface IDalamudPlugin : IDisposable
    {
    }
    ```

    Dalamud instantiates your plugin by calling its constructor. Services are injected by declaring them as constructor parameters. Dalamud resolves each parameter from its internal service container at load time.

    Here is a minimal plugin skeleton:

    ```csharp
    using Dalamud.Plugin;
    using Dalamud.Plugin.Services;

    namespace MyPlugin;

    public sealed class MyPlugin : IDalamudPlugin
    {
        private readonly IPluginLog log;

        public MyPlugin(IDalamudPluginInterface pluginInterface, IPluginLog log)
        {
            this.log = log;
            this.log.Information("MyPlugin loaded!");
        }

        public void Dispose()
        {
            this.log.Information("MyPlugin unloaded.");
        }
    }
    ```

    `IDalamudPluginInterface` is always available and gives your plugin access to its configuration directory, the `IUiBuilder` for drawing UI, version information, and IPC helpers.
  </Step>

  <Step title="Inject a service and register a chat command">
    Dalamud provides services through its dependency injection container. Declare a service as a constructor parameter and Dalamud will supply it automatically.

    The following example injects `ICommandManager` and `IChatGui` to register a `/hello` command that prints a message to the chat window:

    ```csharp
    using Dalamud.Game.Command;
    using Dalamud.Plugin;
    using Dalamud.Plugin.Services;

    namespace MyPlugin;

    public sealed class MyPlugin : IDalamudPlugin
    {
        private const string CommandName = "/hello";

        private readonly ICommandManager commandManager;
        private readonly IChatGui chatGui;
        private readonly IPluginLog log;

        public MyPlugin(
            IDalamudPluginInterface pluginInterface,
            ICommandManager commandManager,
            IChatGui chatGui,
            IPluginLog log)
        {
            this.commandManager = commandManager;
            this.chatGui = chatGui;
            this.log = log;

            this.commandManager.AddHandler(CommandName, new CommandInfo(this.OnHelloCommand)
            {
                HelpMessage = "Say hello to the world.",
            });

            this.log.Information("MyPlugin loaded.");
        }

        public void Dispose()
        {
            this.commandManager.RemoveHandler(CommandName);
        }

        private void OnHelloCommand(string command, string args)
        {
            this.chatGui.Print("Hello, Eorzea!");
        }
    }
    ```

    Key services available as constructor parameters include:

    | Service | Purpose |
    |---|---|
    | `IDalamudPluginInterface` | Core plugin interface — config, UI builder, IPC |
    | `IPluginLog` | Structured logging (Serilog-backed) |
    | `IChatGui` | Print to and intercept chat messages |
    | `ICommandManager` | Register and remove `/slash` commands |
    | `IClientState` | Local player, world, login state |
    | `IDataManager` | Excel sheet and game data access |
    | `IFramework` | Per-frame update event and thread scheduling |
    | `IGameInteropProvider` | Hook native game functions |
  </Step>

  <Step title="Build and load your plugin in-game">
    Build your plugin with the .NET CLI or your IDE:

    ```bash
    dotnet build -c Debug
    ```

    The compiled `.dll` lands in `bin/Debug/net10.0-windows/`.

    To load it in-game:

    1. Start FFXIV through XIVLauncher with Dalamud enabled.
    2. Open the Dalamud plugin installer with `/xlplugins`.
    3. Switch to the **Dev Tools** tab and click **Load Dev Plugin**.
    4. Browse to your compiled `.dll` and select it.

    Your plugin loads immediately. Type `/hello` in chat to verify the command works. View plugin logs in the **Dalamud Dev Menu** (`/xldev` → **Log**).

    <Tip>
      Enable **auto-load** in the dev plugin entry so your plugin reloads automatically each time you rebuild.
    </Tip>
  </Step>
</Steps>
